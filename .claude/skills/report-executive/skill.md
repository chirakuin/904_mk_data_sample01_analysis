# エグゼクティブレポート変換

技術レポート（REPORT.md）を、マーケティング部門・経営層向けの概要版（REPORT_EXECUTIVE.md）に変換する。

## 入力

- 技術レポート: `output/<analysis>/REPORT.md` または `other/REPORT.md`
- パス: `/Users/cc/Documents/Code/000_business/030_MK/A03_CDP_analysis/`

## 出力先

- `output/<analysis>/REPORT_EXECUTIVE.md`（分析個別）
- `other/REPORT_EXECUTIVE.md`（全体まとめ）

## 変換方針

### 構成

1. **この分析で何をしたか**: 1-2文で平易に説明
2. **分析対象**: 対象顧客数・期間などを簡潔な表で
3. **主な発見**: 3-5個のポイントに絞り、ビジネスインパクトを強調
4. **推奨アクション**: 具体的な施策提案（部門・タイミング含む）
5. **補足**: 合成データの制約、実データでの期待値

### 文体・表記

- 技術用語を排除し、ビジネス言語に置換
  - 「クラスタリング」→「顧客タイプの自動分類」
  - 「マルコフ連鎖アトリビューション」→「各チャネルの購買貢献度分析」
  - 「BG/NBDモデル」→「顧客の将来購買予測モデル」
  - 「傾向スコアマッチング」→「条件を揃えた比較分析」
  - 「SARIMAX」→「季節性を考慮した予測モデル」
- p値・信頼区間 → 「統計的に確認済み」「信頼度95%」等に言い換え
- 業界略語は日本語で書く（RTD → 缶チューハイ類等）
- ペルソナ表記は「[嗜好]好きの[年齢帯][性別]」で統一
- 金額・人数は具体的な数字を残す（抽象化しすぎない）
- 図表はシンプルなMarkdown表で。複雑なものは省略し要点のみ

### 分量

- A4 1-2枚相当（400-800文字程度）
- 技術レポートの1/3以下の分量を目安に

## 手順

1. 対象の技術レポート（REPORT.md）を読み取る
2. 上記の変換方針に従い、REPORT_EXECUTIVE.md を生成
3. 既存のREPORT_EXECUTIVE.mdがあれば内容を比較し、更新が必要か確認してから上書き
