# CDP分析と広告運用の関係

2026-03-20の議論メモ。

---

## CDP分析は広告運用につながるか

**直接つながる。** 13分析は広告運用の計画→実行→測定→最適化の全サイクルをカバーしている。

### 広告運用フロー × CDP分析の対応

| フェーズ | やること | 該当する分析 |
|---------|---------|------------|
| 1. 顧客理解 | 誰に売るか | C1クラスタリング, C2ペルソナ, C3コホート, C5 LTV予測 |
| 2. メディアプランニング | どこに・いくら投下するか | M1 MMM, P3予算配分最適化, P1需要予測 |
| 3. ターゲティング・クリエイティブ設計 | 何を見せるか | C1クラスタ, C6離脱予測, J3バスケット, P2 NBA |
| 4. 配信・実行 | 実際に配信する | **CDP分析の範囲外**（広告プラットフォームの領域） |
| 5. 効果測定 | 投下した結果どうだったか | J2アトリビューション, M2 ROI予測, M3 CP効果, M4増分効果, J4ファネル |
| 6. 改善 | 次のサイクルに何を変えるか | J1ゴールデンパス, C4セグメント遷移, M1再実行, P3再配分 |

→ 1に戻る（PDCAサイクル）

### 具体的なシナリオ

- **「来期の広告予算3億円、どう配分すべき？」** → M1(MMM)で弾性値推定 → P3で最適配分算出
- **「デジタル広告のターゲットは？」** → C1でクラスタ → C5で高LTV特定 → C6で離脱リスク考慮
- **「LINE配信の改善ポイントは？」** → J4でファネル分析 → J1で成功パターン特定
- **「先月のキャンペーン、効果はあったのか？」** → M3でリフト算出 → M4で増分効果推定
- **「次に何をすべき？」** → P2(NBA)が顧客ごとに推薦

## 全自動化に足りないもの

分析の自動化はほぼ可能。足りないのは「分析結果を施策に変換して実行する」部分。

```
データ収集 → 分析 → 意思決定 → 実行 → 計測 → フィードバック
   ✅         ✅      ❌        ❌      ✅        △
```

### 5つの欠落部分

| # | 欠落 | 優先度 | 説明 |
|---|------|--------|------|
| 1 | **オーケストレーション** | 高 | 13分析を依存関係順に定期実行するスケジューラー（Airflow, cron等） |
| 2 | **配信リスト生成器** | 高 | 分析結果→各プラットフォーム入力フォーマットへの変換アダプター |
| 3 | **実行系API連携** | 中 | Google Ads API, Meta Marketing API, LINE Messaging API等 |
| 4 | **フィードバックループ** | 中 | 施策実行後のデータを自動でCDPに戻す仕組み |
| 5 | **ルール管理UI** | 低 | NBAのルールや閾値を非エンジニアが変更できる画面 |

### 最小構成で自動化する場合

1と2だけ作れば「分析→配信リスト生成まで自動、プラットフォームへのアップロードだけ手動」という実用的な状態になる。

## NBAとは何か

**Next Best Action（次善最適施策）**。1to1マーケティングの実行エンジン。

顧客1人に対して「次に何を・どのチャネルで・どんなメッセージで」を決定する。最終出力例:

```csv
customer_id,  action,           channel,  message_template,  priority
UC_ba6e5b70,  coupon_10pct_off, LINE,     summer_beer_promo,  high
UC_3f2a1c88,  cross_sell,       email,    whisky_recommend,   medium
UC_9d4e7b22,  retention,        push,     point_expiry_warn,  high
UC_1a8c3d55,  no_action,        -,        -,                  -
```

NBAは他の分析結果を統合するレイヤーであり、単体では成立しない:

```
C1 クラスタリング   → この顧客はどんな人か
C5 LTV予測         → この顧客の将来価値は
C6 離脱予測         → この顧客は離脱しそうか
J2 アトリビューション → どのチャネルが効果的か
J3 バスケット分析     → 何を買いそうか
M3 CP効果分析       → CPは効果があるか
         ↓
P2 NBA → この顧客に次に何をすべきか
```

## 唯一残る人間の仕事: クリエイティブ制作

全体フローの中で、分析パイプラインがカバーできない唯一の部分:

```
誰に       → クラスタ, LTV, 離脱予測で決定       ✅
何を       → バスケット分析, 商品マスタで決定      ✅
いつ       → 需要予測, コホートで決定             ✅
どこで     → アトリビューションで決定              ✅
いくらで   → MMM, 予算最適化で決定               ✅
何と言うか → テンプレートIDまでは出る              ✅
どう見せるか → コピー文・画像・動画の実制作          ❌ ← ここだけ
```

ただし、クリエイティブ制作の**判断材料**は分析から供給可能:

| 判断材料 | 供給元 |
|---------|--------|
| このセグメントはどんな商品に反応するか | C1 + J3 |
| どんなトーンが効くか（価格訴求 vs 品質訴求） | 顧客属性の価格感度 |
| 過去どのクリエイティブのCTRが高かったか | ad_exposure + digital_behavior_log |
| LINEのどの形式が開封→クリックに効くか | J4ファネル分析 |

「こういうクリエイティブを作るべき」という方向性までは分析で出せるが、「実際に作る」部分が人間（またはAI画像生成等）の仕事として残る。
